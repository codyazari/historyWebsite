The Internet Mini-Series

What is the Internet?

  • The internet is a result of ARPANet, which was built to prevent the loss of communication from nuclear attack
  • The internet consists of independently operated networks which ensure that not one person controls or owns the internet. People just partake in the usage of it and share information

Wires, Cables, and Wifi

  • The internet is like a postal service, using binary bits. Binary bits can be considered the “the atoms of information," which comes in the form of 1s and 0s
  • Bits are sent using light, electricity, and radio
  • How do you send multiple 0s? Use a clock, agree that every second information or lack thereof is sent
  • How is more data sent? Increase bandwith, bitrate, and latency. Sending long distance information utilizes fiber optic cables
  • Radio signal is sent using radio waves

IP Addresses and DNS

  • Internet is network of networks
  • Protocol is a set of rules and standards that everyone agrees upon so all can communicate
  • IP address is a street address to send information to and from
  • IP addresses are organized in a hierarchy, represented in 32 bits
  • IPv4 is now being transitioned to IPv6 with 128 bits to keep up with growing demand
  • DNS associates names and addresses, and are connected to each other in a distributed hierarchy, splitting responsibility up
  • DNS is public and prone to hacking such as DNS Spoofing

Packets, Routing, and Reliability

  • How is data delivered reliably? Data is sent in packets, sometimes taking different paths to the same destination to avoid traffic
  • Routers are traffic managers to insure all packets reach their destination
  • Routes are determined by time, politics, and company relationships
  • TCP manages all sending and receiving requests for packets, and signs when all packages arrive, if some are missing, it will not be signed
  • TCP and Routers are scalable, fault tolerance and redundancy increase reliability and grow and scale without interruption

HTTP and HTML

  • Web browsers use URLs to track down certain pages, all of which is communicated using HTTP
  • HTTP is the language used to communicate between web browsers and servers, mostly consisting of get requests
  • HTML is the language to tell a browser what a page looks like
  • Media uses it’s own URL, so separate get requests are sent for those
  • Sometimes data is sent (forms, documents, signatures, etc), those are post requests
  • Cookies are also sent so websites can identify specific users, ID cards essentially
  • Internet is open and uses plain text and is open for hackers, secure websites prevent this using SSL and TLS, layers of security, HTTPS ensures requests are secure
  • Websites use digital certificates to authenticate their website
